# Content Quality Rules — matrixdestiny.co

## Word counts

| Page type | Minimum | Target |
|---|---|---|
| Arcana entity pages | 1,500 | 2,000 |
| Pillar HUB articles | 2,500 | 3,500 |
| Emotional state articles | 1,200 | 1,800 |
| Compatibility pages | 1,400 | 2,000 |
| FAQ / terminal pages | 800 | 1,200 |

## Zero tolerance list

Never publish these phrases. No exceptions.

**Filler phrases:**
- "it is important to note"
- "in conclusion"
- "deeply transformative"
- "this powerful archetype"
- "cosmic energy"
- "universal vibration"
- "the universe wants you to"
- "spiritual awakening journey"
- "ancient wisdom"
- "sacred teachings"
- "divine guidance"
- "mystical forces"

**Structural violations:**
- Generic polarity: never "positive" or "negative" without a behavioral mechanism
- Thin FAQ answers: minimum 3 sentences, target 4–5
- Sections that repeat each other in different words
- Claims without a specific psychological or behavioral mechanism
- Intros that define the arcana name instead of resolving the user's emotional state

## Required elements on every arcana page

Every arcana page must have all of these before publishing:

- [ ] H1 targeting an emotional problem — NOT just the arcana name
- [ ] Opening paragraph naming the psychological function (not the archetype mythology)
- [ ] "What [arcana] describes" section — psychological mechanism, not symbolic history
- [ ] Active vs. blocked section — minimum 4 behavioral examples each
  - For two-pole blocks: both poles described with 3+ behavioral bullets each
- [ ] Three life domains: professional, relationships, karmic pattern
  - Each domain: minimum 2 sentences with specific behavioral description
- [ ] Entity relationship table: IS-A, USED-FOR, WORKS-WITH, CONFLICTS-WITH, CAUSES
- [ ] FAQ block: 5 questions minimum, each answer 3+ sentences
- [ ] Internal navigation: prev arcana / all arcana / next arcana
- [ ] Calculator CTA

## Required elements on every pillar HUB page

- [ ] H1 targeting the primary emotional problem cluster
- [ ] Introduction: what the cluster is and why it matters to this user
- [ ] Complete definition of the central entity
- [ ] Minimum 3 sections covering distinct sub-questions
- [ ] Links to all spoke pages within the cluster
- [ ] Entity relationship table
- [ ] FAQ block: 5 questions minimum
- [ ] Calculator CTA

## Review process

1. Build one page as sample — first arcana in any new language or page type
2. Stop after the sample. Do not build remaining pages.
3. Wait for explicit approval before continuing.
4. Do not commit or push until approval is given.

Exception: if a batch was explicitly pre-approved, build the full batch.

## Behavioral mechanism requirement

Every psychological claim must include a behavioral mechanism — the specific action or pattern the claim produces.

**Bad:**
> "Arcana 1 blocked causes problems with authority."

**Good:**
> "Arcana 1 blocked causes dependence on external validation — the person knows what they want but postpones action until someone more 'experienced' confirms the choice."

If you cannot name the behavioral mechanism, the claim is not ready to publish.

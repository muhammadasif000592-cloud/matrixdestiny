# Query-State Model — matrixdestiny.co

## Core principle

Every page resolves a user's emotional or interpretive state — not a keyword.

A keyword is what the user typed. A query state is what the user actually needs resolved. These are different, and content that targets the keyword without resolving the state fails both the user and the ranking.

> "Arcana 8" is a keyword.
> "Why do the same situations keep happening to me" is a query state.

The page that ranks for "arcana 8" long-term is the one that resolves the actual state behind the search.

## Query-state definition

A query state is:
- A specific emotional condition the user is currently in
- A question the user is trying to answer about their own life
- An interpretation the user is trying to validate or refute
- A pattern the user is trying to name

Every page targets exactly ONE primary query state. Secondary states are served within the content, but the H1, meta description, and opening section resolve the primary state.

## How to identify the query state for a page

Ask: "What is the person actually experiencing when they search for this?"

| What they searched | What they actually need |
|---|---|
| arcana 8 meaning | Why the same situations keep happening |
| arcana 11 blocked | Why I lose emotional control or go numb |
| matrix of destiny compatibility | Whether this relationship makes sense |
| what is matrix of destiny | Whether this system has real explanatory power |
| arcana 4 emperor | Why I can't hold a boundary without guilt or anger |

## Required emotional state clusters

These clusters must be represented across the content. Every cluster needs at minimum: 1 pillar HUB or arcana page that resolves the primary state.

### Decision and agency cluster
- Decisional paralysis — "I know what I want but I can't act on it"
- Chronic indecision — "I change direction before anything completes"
- Loss of authorship — "Life happens to me, not through me"
- Relevant arcana: 1, 7, 22

### Emotional regulation cluster
- Emotional explosion / loss of control — "I react in ways I regret"
- Emotional suppression / numbness — "I don't feel anything anymore"
- Anxiety without clear cause — "I'm afraid and I don't know why"
- Relevant arcana: 11, 15, 18

### Relationship and commitment cluster
- Commitment avoidance — "I always flee when things get real"
- Choosing wrong partners — "I keep attracting the same type"
- Boundary failure — "I can't say no without guilt or aggression"
- Relevant arcana: 4, 6, 8, 15

### Pattern repetition cluster
- Same situations repeating — "Why does this keep happening to me"
- Karmic loops — "I've been through this exact thing before"
- Stuck in a phase — "Nothing moves forward"
- Relevant arcana: 8, 10, 12, 13

### Purpose and meaning cluster
- Wasted life feeling — "I'm going through the motions"
- Hollow success — "I achieved what I wanted and felt nothing"
- Calling not answered — "I know what matters but I'm not doing it"
- Relevant arcana: 9, 17, 20, 21

### Visibility and identity cluster

- Shame around being seen — "I'm afraid to take up space"
- Impostor pattern — "I don't believe I'm really good enough"
- Relevant arcana: 5, 19, 22

### Transition and change cluster

## Query state vs. keyword targeting

| Level | Example | Quality |
|---|---|---|
| Keyword only | "arcana 18" | Fails — no state resolution |
| Keyword + arcana type | "arcana 18 moon meaning" | Weak — still not a state |
| Keyword + question | "arcana 18 anxiety meaning" | Better — implies a state |
| Full query state | "Why you feel anxious without cause and imagine the worst" | Correct — resolves the state |

## H1 formula

Every arcana page H1 follows this formula:

> **Arcana [N] — [Name]: [Emotional problem statement]**

The emotional problem statement must:
- Name a specific, recognizable condition
- Use second-person ("you") — this is the user's problem, not a definition
- Describe the experience, not the arcana's symbolic meaning

**Bad H1 examples:**
- "Arcana 8 — Justice: Meaning and Interpretation" — no state
- "Arcana 8 — Justice: The Archetype of Cause and Effect" — symbolic, not experiential
- "Arcana 8 — Justice: Balance and Fairness in Your Chart" — keyword stuffing

**Good H1 examples:**
- "Arcana 8 — Justice: Why the Same Situations Keep Happening to You" — resolves pattern repetition state
- "Arcana 11 — Strength: Why You Lose Control of Your Emotions or Suppress Them Completely" — resolves regulation state
- "Arcana 12 — The Hanged Man: Why Your Life Feels Stuck and How to Use the Pause Productively" — resolves stagnation state

## Meta description formula

The meta description extends the H1 by naming the psychological mechanism and the specific benefit of reading the page.

Formula:
> "[Arcana N] in the Matrix of Destiny represents [psychological function]. Discover [specific thing user will learn about their pattern]."

**Bad meta:**
- "Learn about Arcana 8 meaning in destiny matrix. Justice arcana interpretation and analysis." — keyword stuffing, no state

**Good meta:**
- "Arcana 8 in the Matrix of Destiny represents the capacity to read cause and effect with accuracy and take precise accountability for your own contribution. Discover why you keep attracting the same patterns and how to break the cycle." — resolves the state

## FAQ query states

Each FAQ question in the 5-question block must address a distinct query state within the arcana's cluster. No two FAQ questions should resolve the same state.

Minimum variety across the 5 questions:
1. "What does it mean to have [arcana] in the Matrix of Destiny?" — orientation state
2. "Why does blocked [arcana] cause [specific problem]?" — mechanism state
3. A nuanced distinction question — "What is the difference between X and Y in this arcana?" — precision state
4. A scope or application question — "Does [arcana] relate to [specific domain]?" — scope state
5. A practical development question — "How do you work with [arcana] when it is blocked?" — action state
